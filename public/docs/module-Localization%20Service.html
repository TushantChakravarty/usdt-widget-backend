(static) getText(lang, screen, key) → {Promise.<Object>}

Service to retrieve text according to the user's app language. The `getText` service takes three parameters:
Parameters:
Name Type Description
lang string The language code (e.g., "en" for English, "hi" for Hindi).
screen string The screen name to retrieve text for (e.g., "home", "login", "contest").
key string The specific key to retrieve text for (optional).
Source:
  • services/localization.service.js, line 120
Returns:
- A promise that resolves to the retrieved text object.Returns a blank object if the language, screen, or key is not found.
Type
Promise.<Object>
Example
Inside your code, you can use the `getText` service as follows:
const lang = "en";
const screen = "home";
const key = "heading";
const text = await getText(lang, screen, key);
console.log(text); // Output: { heading: "Welcome" }

If you want to retrieve all pages with key-value pairs for a language, you can call the service with just the language parameter:
const lang = "en";
const allPagesText = await getText(lang);
console.log(allPagesText); // Output: { home: {...}, login: {...}, contest: {...}, ... }

If you want to retrieve all key-value pairs of a specific screen for a language, you can call the service with both the language and screen parameters:
const lang = "en";
const screen = "login";
const loginScreenText = await getText(lang, screen);
console.log(loginScreenText); // Output: { heading: "Login", btn: "Submit", ... }
